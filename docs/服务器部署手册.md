## 1. 三种「管整机」的身份（勿混）

| 身份 | 干什么 | 谁来当 |
|------|--------|--------|
| **服务器部署者** | 买云主机、装系统、拉代码、配域名与证书的人 | 店老板自己，或请的技术人员 |
| **服务器管理者** | 日常改站点名称、备案展示、服务器主页等（进「服务器设置」） | 部署者指定的普通登录账号；**无技术的人可以当** |
| **超级管理员** | 进总后台 `/admin/`，权限极大 | **只给有技术、有经验的维护人员** |

原则（定稿）：

> 普通人（无技术）默认**不当**超级管理员。  
> 超级管理员只给会维护、懂后果的人。  
> 日常一律走「服务器设置」；总后台只用于安装、救急、指定管理者。

## 2. 超级管理员说明（仅写在本手册）

### 2.1 它有多大权限？

对这台野草网站来说，超级管理员接近「上帝」：能改用户身份、订单、店铺、支付相关配置等（凡挂在总后台里的）。  
**乱点可能弄坏全站数据。** 因此：

- 不要写进用户操作手册  
- 不要交给只负责点菜/管店、不懂技术的人当日常账号  
- 密码单独保管，不要和店主日常登录混用（建议）

### 2.2 什么时候才需要它？

例如：

- 首次安装后创建网站超级管理员  
- **指定 / 取消「服务器管理者」**（在用户编辑页勾选「服务器管理者」）  
- 救急纠错（账号锁死、数据异常等）  
- 技术维护人员排查问题  

日常改备案展示、站点名称、服务器主页 → 用 **「服务器设置」**（`/server-settings/`），不必进总后台。

### 2.3 如何创建超级管理员（命令行）

在服务器项目目录、已激活虚拟环境后：

```text
python manage.py createsuperuser
```

按提示设置用户名和密码。然后浏览器打开：

```text
https://你的域名/admin/
```

进入总后台前，页面会有**高风险警告**；确认自己是技术维护人员再继续操作。

### 2.4 如何指定服务器管理者

1. 用超级管理员登录 `/admin/`  
2. 打开「用户」→ 点开目标账号（一般是店主）  
3. 在「用户身份」中勾选 **服务器管理者** → 保存  

或在服务器执行（与是否官方小店无关）：

```text
python manage.py mark_server_owner --username=用户名
```

被指定的人登录网站后，菜单可见 **「服务器设置」**。

### 2.5 总后台与 Linux 登录不是一回事

云主机上的系统用户（如 `admin`）≠ 网站超级管理员。  
能 SSH 登录服务器，不等于能进 `/admin/`；反之亦然。

## 3. 安装与更新（摘要）

1. 服务器准备 Python、Nginx、HTTPS 等（细项见私人部署指南或云厂商文档）。  
2. 从公开仓库克隆或 `git pull` 更新代码。  
3. **`git pull` 不会覆盖**服务器上的 `.env` 与业务数据库（私人配置得以保留）。  
4. pull 后按需执行：`migrate`、`collectstatic`、重启进程。  
5. 配置 `.env`（网站签名钥匙、`YECAO_DEBUG=0`、域名等），参考仓库 `.env.example`。  
6. 创建超级管理员 → 指定服务器管理者 → 管理者在「服务器设置」填写备案与站点品牌。

概念分界（正式服务器 vs 私人体验机）见：`docs/正式服务器与体验机分界.md`。
