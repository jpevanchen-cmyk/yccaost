## 3. Python 依赖（`requirements.txt`）

安装命令（在项目根、已建好 `.venv` 时）：

```text
.\.venv\Scripts\python.exe -m pip install -r requirements.txt
```

| 包 | 版本约束（以 requirements 为准） | 许可（摘要） | 谁需要 | 做什么 | 官方 / 可信来源 | 怎么验证 | 能否换低配/高配 |
|----|----------------------------------|--------------|--------|--------|-----------------|----------|-----------------|
| Django | 5.2.15 | BSD-3-Clause | 全站 | 网页框架 | https://pypi.org/project/Django/ | `python -c "import django; print(django.get_version())"` | 勿随意大跨版本 |
| asgiref | 3.11.1 | BSD-3-Clause | Django | 异步接口支撑 | https://pypi.org/project/asgiref/ | 随 Django 安装 | 跟 Django 兼容即可 |
| sqlparse | 0.5.5 | BSD | Django | SQL 解析 | https://pypi.org/project/sqlparse/ | 随 Django | 跟 Django |
| tzdata | 2026.2 | Apache-2.0 | 时区 | Windows 等环境时区数据 | https://pypi.org/project/tzdata/ | 页面北京时间正常即可 | 可更新 |
| fpdf2 | 2.8.3 | **LGPL-3.0** | 桌贴导出 | 生成桌贴 PDF | https://pypi.org/project/fpdf2/ | 卖家导出桌贴 PDF | 小版本可试；许可见第三方表 |
| qrcode | 8.2（含 pil 额外） | BSD | 二维码 | 工作台 / 桌码等 | https://pypi.org/project/qrcode/ | 后台生成二维码 | 小版本可试 |
| Pillow | ≥10.0.0 | MIT-CMU | 图像 | 二维码等图像处理 | https://pypi.org/project/Pillow/ | 同上 | 建议 ≥10 |

### SQLite 版本（Django 5.2 硬要求）

| 项 | 说明 |
|----|------|
| 为何要管 | Django 5.2 要求 **SQLite ≥ 3.31**；部分云服务器系统自带仍是 3.26 等旧版，直接 `migrate` 会失败 |
| 程序怎么做 | 启动 / 管理命令前**先检测**；够用则用系统自带；不够则尝试使用可选兼容包；都没有则**白话退出**并指向本说明 |
| 标准解法（全体部署者同一套） | 在虚拟环境中执行：`pip install -r requirements-sqlite-compat.txt`（内含 `pysqlite3-binary`），再重新 `migrate` / 启动 |
| 自检命令 | `python manage.py check_sqlite` |
| 不是偷偷只修某一台 | 检测与提示写在仓库里；可选包装在独立依赖文件，避免强迫所有环境必装 |

新增包后须同步更新本文件、`requirements.txt` 与 `docs/THIRD_PARTY_LICENSES.md`。

## 4. 运行时数据与日志（非 pip 包）

| 项 | 位置 | 说明 | 进 Git？ |
|----|------|------|----------|

## 6. 常用启动与检查

```text
.\.venv\Scripts\python.exe manage.py migrate
.\.venv\Scripts\python.exe manage.py runserver
.\.venv\Scripts\python.exe manage.py check
```

手机局域网测试：`runserver 0.0.0.0:8000`，并用 `ipconfig` 查本机 IP。

本地地址备忘见根目录 `服务器启动命令以及网址等.txt`（不进 Git）。
