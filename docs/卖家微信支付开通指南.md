# 野草系统 · 卖家微信支付开通指南

> **写给谁看**：打算在自己店铺里接**真微信支付**的卖家。  
> **说明**：野草系统**不替你收钱**；钱由微信支付直接进入**你自己申请的商户账户**。下面按时间顺序，从准备到试营业一步步写。  
> **和演示支付的区别**：「演示支付（模拟）」不扣真钱，只用于体验流程；真微信支付会**真实扣款**，正式营业前请关闭演示支付。

---

## 目前代码实际支持到哪里（2026-07-15 核对）

商户平台显示某项“已开通”，只代表微信允许该商户申请使用，**不代表野草代码已经支持**。

| 微信支付产品 / 能力 | 野草当前状态 |
|:---|:---|
| **Native 支付** | ✅ 已有代码：生成扫码链接与二维码、查询支付结果、校验并处理微信通知、记录支付流水；**尚未完成新商户真实小额付款验证** |
| **JSAPI 支付** | ❌ 尚未实现 |
| **刷脸支付** | ❌ 尚未实现 |
| **模拟支付** | ✅ 已实现，仅供演示，不扣真钱 |
| **现金支付** | ✅ 已实现 |
| **通用支付结算 / 分账** | ❌ 尚未完成；现有支付仍与餐饮订单结合较深，通用通道已记入第一阶段待完善 |

**重要接口风险**：野草当前 Native 代码使用微信支付 **APIv2**（商户号 + AppID + APIv2 密钥）。微信当前面向新商户的官方 Native 接入文档以 **APIv3** 为主。即使商户平台显示 Native 已开通，也不能据此认定现有 APIv2 代码一定可用；须先核实新商户是否还能配置并使用 APIv2，再做真实小额测试。若新商户只支持 APIv3，野草需要另行升级支付接入代码。

## 开通前要心里有数（一张总览表）

| 顺序 | 你要做的事 | 大概在哪完成 |
|:---:|:---|:---|
| 1 | 有合法经营主体（如个体工商户营业执照） | 线下 / 政务 |
| 2 | 申请微信支付商户号 | 微信支付商户平台 |
| 3 | 拿到商户号、AppID、APIv2 密钥 | 商户平台 |
| 4 | 有域名 + 云服务器 + 安全网址（HTTPS） | 云服务商 |
| 5 | 把野草系统部署到服务器上 | 你的服务器 |
| 6 | 在野草「支付设置」里填写并保存 | 店铺管理 → 支付 |
| 7 | 用低价商品试付一笔 | 买家端 |
| 8 | 关闭演示支付，正式营业 | 店铺管理 → 支付 |

**本地电脑演示**（同一 WiFi 手机访问）可以一直用「演示支付」，**不必**完成 4～7 也能开发、体验下单流程。  
**要让别人在互联网上真付钱**，一般需要做完 **2～7**。

## 第一步：准备经营主体材料

微信支付要求你是**真实经营者**，个人随便一个微信号通常**不能直接**当商户。

1. 准备好**个体工商户营业执照**（或公司执照，按你实际主体来）。  
2. 执照上的名称、法人、结算银行卡等信息，后面在网上填申请表时要**和执照一致**。  
3. 准备一个**用于收款的银行账户**（一般是对公或法人银行卡，以微信审核要求为准）。  
4. 准备一个**常用手机号**和**邮箱**，用于接收审核通知。

> 若你还没有执照：可以先在本地用「演示支付」熟悉野草；执照和商户号可以**并行办理**，不耽误你研究系统。

---

## 第二步：申请微信支付商户号

1. 用电脑浏览器打开微信支付商户平台：  
   **https://pay.weixin.qq.com**
2. 选择「成为商家」或「入驻」类入口（页面文案可能更新，找**申请商户号**即可）。  
3. 按页面提示选择主体类型（如**个体工商户**）。  
4. 上传营业执照照片、填写店铺名称、经营类目（外卖餐饮选相近类目即可，以平台选项为准）。  
5. 填写结算银行卡、法人身份等信息。  
6. 提交后等待**审核**。  
   - 快则几天，慢则一两周，可能要补材料。  
   - 审核通过后，你会获得**商户号**（一串数字，野草里叫「微信商户号」）。

**请记下：**

- **商户号**（mch_id，一般是 10 位数字）  
- 登录商户平台用的账号（自行保管，不要交给别人）

---

## 第三步：开通「Native 扫码支付」并绑定 AppID

野草第一阶段用的是：**网页下单 → 出二维码 → 买家微信扫一扫付款**（技术上叫 Native 扫码）。

### 3.1 确认已开通扫码类产品

1. 登录 **https://pay.weixin.qq.com**  
2. 进入「产品中心」或「我的产品」（名称以页面为准）。  
3. 找到 **Native 支付**（或「扫码支付」）并确认状态为**已开通**。  
4. 若未开通，按页面申请开通。

### 3.2 准备 AppID 并与商户号绑定

支付时除了商户号，还需要 **AppID**（可理解为微信里识别你这套应用的一串 ID）。

**2026-07-15 实测提醒**：

- 微信当前要求商户号关联符合要求、通过认证的 AppID；只有商户号、没有可绑定的 AppID，仍不能发起 Native 支付。
- 开发者本次办理时，认证页面显示审核服务费 **300 元**。该费用由微信/审核服务机构收取，不是野草收费，也不是单纯“开通 Native 产品”的费用。
- 认证通常有有效期，后续重新认证可能再次收费；认证失败是否退费、有效期和金额均以申请时微信官方页面为准，不在野草中写死承诺。
- 不要把网上“随便注册一个免费个人小程序即可绑定支付”的说法当成可靠方案；应以微信商户平台实际允许绑定的账号类型和认证状态为准。

常见做法（任选一种，以你能办到的为准）：

| 方式 | 说明 |
|:---|:---|
| **微信公众号** | 若有**已认证的服务号**，可在公众号后台看到 AppID，再到商户平台里与商户号**关联**。 |
| **微信开放平台** | 注册开放平台账号，创建「网站应用」或相关应用，获得 AppID，再与商户号绑定。 |
| **小程序** | 若有小程序，也可用小程序 AppID 与商户号关联（野草当前网页扫码以 Native 为主，多数卖家用公众号或开放平台 AppID）。 |

**操作要点：**

1. 在商户平台找到 **「AppID 账号管理」** 或 **「产品设置 → 开发配置」** 等类似菜单。  
2. 准备符合页面要求并已完成认证的微信账号，取得 AppID。
3. 在商户平台发起 AppID 与商户号的关联申请。
4. 登录该 AppID 所属的公众平台或开放平台，确认授权绑定。
5. 复制 **AppID**（一般以 `wx` 开头），后面要填进野草。

> 若 AppID 和商户号**没绑定**，野草里即使填对了数字，微信也会拒绝下单。

### 3.3 设置 APIv2 密钥（重要）

1. 在商户平台打开 **「账户中心」→「API 安全」**（或类似名称）。  
2. 找到 **APIv2 密钥**（32 位，需自己设置）。  
3. 点击「设置」或「修改」，按提示设置一串 **32 位密钥**（字母+数字，自己牢记或安全保存）。  
4. **这不是野草自动生成的**；必须你在微信商户平台设置，再复制到野草里。

**安全提醒：**

- APIv2 密钥相当于收钱大门的钥匙，**不要**发微信群、不要截图外传。  
- 只填在**你自己店铺服务器**上的野草「支付设置」里。  
- 浏览器可能误把别的密码自动填进「密钥」框，若你还没申请商户，这一格应**留空**。

## 第四步：准备公网访问（域名 + 服务器 + HTTPS）

买家在互联网上访问你的店、微信把「已付款」通知发给野草，都需要一个**公网上能打开的网址**，且微信正式环境一般要求 **HTTPS**（地址以 `https://` 开头）。

### 4.1 域名

1. 你已有域名的话，在域名服务商处把域名**解析**到云服务器的 IP。  
2. 若服务器在国内，按云厂商要求完成**备案**（备案常需数周，宜尽早办）。

### 4.2 云服务器

1. 租用一台云服务器（阿里云、腾讯云等均可）。  
2. 在服务器上安装运行野草所需环境（Python、数据库等，按项目部署说明操作）。  
3. 把野草程序部署上去，确保用浏览器能打开你的店铺名录页。

### 4.3 HTTPS 证书

1. 在云厂商申请免费 SSL 证书，或用 Let’s Encrypt 等方式。  
2. 配置好后，用 **https://你的域名** 能正常打开网站。  
3. 记下这个地址，后面填「店铺公网网址」时要用，例如：  
   `https://shop.example.com`  
   **不要**在末尾多加斜杠，野草会自动拼后面的路径。

### 4.4 微信通知地址（了解即可）

保存公网网址后，野草使用的微信付款通知地址为：

```text
https://你的域名/pay/wechat/notify/
```

你**一般不用手动**去微信里填这个地址（Native 下单时野草会带给微信）。  
但要保证：**公网 + HTTPS + 野草正在运行**，微信才能访问到这个地址。

> **暂时没有公网时**：野草支付页会**定时向微信查单**，有时也能变「已支付」，但不如正式通知稳定；**试营业、正式收款建议务必配好 HTTPS 公网**。

## 第五步：在野草店铺管理里填写支付设置

1. 用**卖家账号**登录你的野草网站。  
2. 进入 **店铺管理 → 支付**（地址形如：`/seller-panel/payment/`）。  
3. 按下面说明逐项填写并**保存**。

| 野草里的设置项 | 填什么 |
|:---|:---|
| **开启微信支付（真扣款）** | 试营业前可先不勾；参数都准备好后再勾选。 |
| **开启模拟支付** | 开发、演示时可勾；**正式营业务必取消勾选**。 |
| **开启现金支付** | 按需；与微信无关。 |
| **示范店模式** | 当前无实际控制或展示作用，已列入删除待办；无需依赖它。要同时提供模拟与真微信，直接分别开启两种支付（仅官方/正式店可开真微信）。 |
| **微信商户号** | 第二步拿到的商户号（纯数字）。 |
| **微信 AppID** | 第三步绑定好的 AppID（`wx` 开头）。 |
| **微信 APIv2 密钥** | 第三步在商户平台设置的 32 位密钥。 |
| **店铺公网网址** | 第四步的 `https://你的域名`（不要末尾 `/`）。 |

4. 点 **「保存支付设置」**。  
5. 页面下方会显示拼好的**微信异步通知地址**，核对是否为：  
   `https://你的域名/pay/wechat/notify/`

**检查清单：**

- [ ] 三项微信参数都已填，不是浏览器乱填的密码  
- [ ] 公网网址能在手机 **4G/5G 网络**（关 WiFi）下打开  
- [ ] 已勾选「开启微信支付」  
- [ ] 正式店已关闭「开启模拟支付」

## 第六步：用低价商品做一笔真支付测试

1. 在 **菜品管理** 里上架一个**测试商品**（如名称写「支付测试专用 0.1 元」，价格 **0.1 元**；若微信不允许 0.1，可改为 **1 元**）。  
2. 用**买家账号**（或另注册一个）在手机上下单。  
3. 进入支付页，选择 **「微信支付」**（不要选演示支付）。  
4. 用微信扫页面上的二维码，完成付款（会**真扣款**）。  
5. 正常情况下：  
   - 几秒内订单变为 **「已支付」**；  
   - 卖家在 **订单管理** 里能看到待备货。  
6. 登录 **微信支付商户平台**，在「交易中心」查看是否收到该笔款项。

**若付了钱但订单一直「待支付」：**

- 看公网网址、HTTPS 是否配置正确；  
- 看 APIv2 密钥、商户号、AppID 是否抄错；  
- 看 AppID 是否与商户号已绑定；  
- 稍等片刻（野草也会自动查单）；  
- 仍不行：查服务器日志，或暂时联系帮你部署的人排查。

---

## 第七步：正式营业前最后确认

1. **关闭「开启模拟支付」**，避免顾客误点演示支付以为已付款。  
2. 测试商品可下架或改价。  
3. 再次用**小额真付**走一遍：下单 → 扫码 → 备货 → 派单 → 送达。  
4. 阅读《野草系统 · 核心规则速查手册》**附录 G** 中关于退款、对账的说明；退款需在野草或商户平台按规定操作。  
5. 上线前对照《野草系统 · 正式上线前清查备忘录》做安全检查（关闭调试模式、强密码等）。

## 常见问题（白话）

**问：野草会不会经过我的手再转给卖家？**  
答：不会。第一阶段钱从买家微信直接进入**你的商户账户**，野草只提供技术工具。

**问：APIv2 密钥框里怎么已经有圆点？**  
答：野草**不会**自动生成。多半是**浏览器自动填了别的密码**，或你以前保存过。没商户前应**清空保存**。

**问：没有执照能先接微信吗？**  
答：不能正式申请商户。可先用「演示支付」体验野草。

**问：只有局域网、没有云服务器能真收款吗？**  
答：不稳定，也不适合给外人用。正式收款请用 **HTTPS 公网**。

**问：官方小店能同时开模拟和真微信吗？**
答：可以，直接分别开启两种支付方式，不需要勾选「示范店模式」。体验店仍禁止真微信；正式营业前应关闭模拟支付。

**问：以后还能接支付宝吗？**  
答：野草已预留多种支付方式位置；支付宝等开通后会在支付页增加选项，流程与微信类似（各自申请商户、各自填参数）。
