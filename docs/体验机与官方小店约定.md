# 体验机与官方小店约定（定稿 · 仅私人体验部署）

> **读者**：开发者 / 智能体  
> **定稿日期**：2026-07-14；**2026-07-16 澄清**：本文件**只**描述开发者私人体验机，不是正式服务器通用规则。  
> **通用正本**：`docs/正式服务器与体验机分界.md`（正式服务器 / 服务器部署者 / 服务器设置）。  
> **服务器举例**：阿里云轻量 2 核 / 2G / 40G（Alibaba Cloud Linux）——以下默认上限按此量级。

---

## 0. 先读：别和正式服务器混

| 本文件（私人体验机） | 正式服务器（通用产品） |
|----------------------|------------------------|
| 「官方小店」「正式保留号」「体验日清」 | **没有**这些系统必有概念 |
| 为开放测试、收真实反馈 | 用户自己部署野草后的正常服务器 |
| 完善功能时**不得**把这里的叫法写进通用权限/主流程 | 以服务器部署者 + 服务器设置为准 |

**这不是改路线，只是先上线做真实测试。** 完善野草仍按手册已订路线一步步来。

---

## 1. 节点定位（仅本私人机）

- 本机可为：**功能体验机** + 开发者自己起名的「官方小店」（私人商铺测试用）。  
- **不**做成「对外开放商家入驻平台」：公网默认走体验开店/体验注册。  
- 服务器属于部署者本人，不是野草平台中央入驻站。

---

## 2. 官方小店 vs 体验（仅体验机开关打开时）

| 类型 | 说明 |
|------|------|
| **官方小店** | 私人体验机上标记的自用店；**不占**体验店/体验账号每日额度；**不参与**每日 24 点清空；可配置**真实支付** |
| **体验账号 / 体验店** | 公网注册与体验开店产生；可走完整流程；**禁止真实支付**；数据不能长期保留 |

注册提示大意：体验注册、禁真付、真付请到官方小店、每天 24 点清空。

---

## 3. 默认额度（可配置）

| 项 | 默认值 | 备注 |
|----|--------|------|
| 每日体验**店铺**上限 | **15** | 官方小店不计入 |
| 每日体验**账号**上限（含开店账号） | **30** | 正式保留号不计入 |
| 同时在线上限 | **10** | 超限提示繁忙（不做排队） |
| 达日额度文案 | 今日体验用户数量已达上限，明日请早！ | 可区分开店满 / 注册满 |

---

## 4. 行为摘要

1. 部署时开通并标记官方小店（私人测试用）；公网以体验开通为主。  
2. 体验开店必须保留完整流程。  
3. 体验禁真付；真付只在官方小店。  
4. 每天 24:00（北京时间）清空体验数据；官方小店保留。  
5. 同时在线超限提示繁忙。  
6. 顶满或异常尽量邮件通知（须冷却）。

**服务器设置权限**与官方小店/正式保留号**无关**：管理者只由超级管理员或命令指定（见分界文档）。

---

## 5. 实现开关与私人状态存放

- `YECAO_EXPERIENCE_SITE=1` 时本约定生效；未开启则接近普通正式服务器行为。  
- **开关与额度**宜写在服务器 `.env`（不进 Git）。  
- **哪家是官方店、谁体验/保留**存在业务库里（不进 Git）。  
- `git pull` 只更新代码，**不会**清掉上述私人状态（详见分界文档第 4 节）。

| 环境变量 | 默认 | 含义 |
|----------|------|------|
| `YECAO_EXPERIENCE_SITE=1` | 关 | 开启体验机规则 |
| `YECAO_EXPERIENCE_DAILY_SHOPS` | 15 | 每日体验店上限 |
| `YECAO_EXPERIENCE_DAILY_ACCOUNTS` | 30 | 每日体验账号上限 |
| `YECAO_EXPERIENCE_CONCURRENT` | 10 | 同时在线上限 |
| `YECAO_EXPERIENCE_NOTIFY_EMAIL` | 空 | 通知邮箱 |

### 管理命令（体验机）

```text
python manage.py mark_official_shop --seller_id=店主用户名
python manage.py purge_experience_data
```

指定服务器管理者（通用，与是否官方小店无关）：

```text
python manage.py mark_server_owner --username=用户名
```

## 6. 与部署的关系

- 云上宜用空库首次安装；之后用 pull 更新，保留 `.env` 与数据库。  
- 网站签名钥匙、体验开关在服务器 `.env`。  
- 微信密钥只在店内后台填写，不进代码。  
- 细项见本地 `docs/正式上线部署指南.md`（不进 Git）与 E 盘进度。
