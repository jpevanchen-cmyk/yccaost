# 体验机与官方小店约定（定稿）

> **读者**：开发者 / 智能体  
> **定稿日期**：2026-07-14  
> **适用**：本仓库可作「体验服务器 + 拥有者官方小店」部署；其它纯自用节点可将体验模式关闭。  
> **服务器举例**：阿里云轻量 2 核 / 2G / 40G（Alibaba Cloud Linux）——以下默认上限按此量级。

---

## 0. 与手册路线的关系（必须牢记）

**这不是改路线，只是先上线做真实测试。**

| 是什么 | 不是什么 |
|--------|----------|
| 先把节点部署上线，用**官方小店**做真实支付与真实运转测试 | 另开一条与手册并行的「体验机产品线」 |
| 开放体验，收集**真实反馈**，再按手册已订路线一步步完善 | 因体验机安排而改写手册优先级、阶段目标或主路线 |
| 云上部署 + 体验机规则，服务「上线试跑 → 听反馈 → 按手册完善」 | 用体验机替代手册里的完善路径（退款、隐私、A.15 等仍按原订顺序） |

智能体与其它窗口：**完善野草仍以《核心规则速查手册》与已定开发方案为准**；体验机/官方小店只是当前部署形态与测试手段，不得据此擅自改路线。

---

## 1. 节点定位

- 本机可为：**功能体验机** + **拥有者自己的官方小店（私人商铺）**。  
- **不**做成「对外开放商家入驻平台」：公网默认走**体验开店/体验注册**；正式店只保留拥有者那一家（或少数官方店）。  
- 服务器属于**拥有者**，不是野草平台中央入驻站（与 A.10 / 所有权约定一致）。

---

## 2. 官方小店 vs 体验

| 类型 | 说明 |
|------|------|
| **官方小店** | 部署时开通并标记为正式/保留；**不占**体验店/体验账号每日额度；**不参与**每日 24 点清空；可配置**真实支付**做测试与自用 |
| **体验账号 / 体验店** | 公网注册与体验开店产生；可走**完整开店与经营流程**；**禁止真实支付**；数据**不能长期保留** |

注册时须醒目提示（大意）：

> 这是一次**体验注册**，可体验完整功能，但**无法开通真实支付**。若要测试真实支付，请到**官方小店**。所有体验帐户将在**每天 24 点**自动清空重置。

---

## 3. 默认额度（可配置）

| 项 | 默认值 | 备注 |
|----|--------|------|
| 每日体验**店铺**上限 | **15** | 官方小店不计入 |
| 每日体验**账号**上限（含开店账号） | **30** | 官方相关正式号不计入 |
| 同时在线上限 | **10** | 第一版：超限提示「当前繁忙，请稍后再试」（不做排队） |
| 达日额度文案 | 今日体验用户数量已达上限，明日请早！ | 可区分「开店满」与「注册满」 |

---

## 4. 行为摘要

1. **正式开店**：部署时做一次（或受控通道）；之后公网以体验开通为主。  
2. **体验开店必须保留**（完整流程）。  
3. **体验禁真付**；真付只在官方小店。  
4. **每天 24:00（北京时间）**清空所有体验帐户及相关体验店/订单等；官方小店保留。  
5. **同时在线**超限：不允许新连接占用名额（提示繁忙）。  
6. **顶满或异常**：尽量邮件通知拥有者（须冷却，避免刷信）。

---

## 5. 实现开关

- 配置项（环境变量 / 设置）开启「体验机模式」后，上述规则生效。  
- 未开启时，行为接近普通单店节点（便于本地开发）。

| 环境变量 / 设置 | 默认 | 含义 |
|------------------|------|------|
| `YECAO_EXPERIENCE_SITE=1` | 关 | 开启体验机规则 |
| `YECAO_EXPERIENCE_DAILY_SHOPS` | 15 | 每日体验店上限 |
| `YECAO_EXPERIENCE_DAILY_ACCOUNTS` | 30 | 每日体验账号上限 |
| `YECAO_EXPERIENCE_CONCURRENT` | 10 | 同时在线上限 |
| `YECAO_EXPERIENCE_NOTIFY_EMAIL` | 空 | 达上限/异常通知邮箱 |

### 管理命令

```text
# 部署后标记自己的正式店（不占额度、不日清、可真付）
python manage.py mark_official_shop --seller_id=店主用户名

# 每天北京时间 0 点清空体验数据（建议挂 cron / 计划任务）
python manage.py purge_experience_data
```

### 代码落点（已落地）

- 用户：`is_experience` / `is_permanent`；店铺：`is_official`
- 注册/开店提示与日额度校验；体验禁真付
- 同时在线繁忙提示；邮件通知（冷却约 30 分钟）
- 日清与官方标记命令如上

## 6. 与部署的关系

- 云上宜用**空库**（勿上传本机数据库）。  
- 网站签名钥匙用环境变量 `DJANGO_SECRET_KEY`（或 `YECAO_SECRET_KEY`）在服务器单独配置；参考根目录 `.env.example`。  
- 微信商户密钥只在官方小店后台填写，不进代码、不进 Git。  
- 部署后标记官方小店；开启体验机模式与通知邮箱。  
- 细项操作见本地 `docs/正式上线部署指南.md`（不进 Git）与 E 盘进度。
