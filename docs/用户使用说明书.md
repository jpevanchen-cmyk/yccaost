# 野草系统 · 用户使用说明书

> **读者**：店主、店员、骑手、买家（最终用户）  
> **最后更新**：2026-07-15
> **说明**：用白话写「能做什么、怎么串起来用」。开发细节见 `开发方案.md` 等开发者文档。  
> **业务硬规则**以仓库《野草系统-核心规则速查手册》为准；本文只讲常用操作。

---

## 1. 这套系统是什么

野草系统是一套**网页**（手机、电脑都能开），帮一家店完成：

- 客人点菜、下单、付款（模拟 / 微信 / 现金等，按店配置）  
- 店主管菜品、营业、桌码、菜单清单、配送费、支付  
- 服务员、后厨、骑手在**店铺工作台**处理现场订单  

第一阶段以**堂食为主、外卖为辅**；每家店自己部署、自己收款，平台不抽佣。

---

## 2. 有哪些「门」和角色

| 角色 | 从哪进 | 能干什么（概要） |
|------|--------|------------------|
| **服务器拥有者** | `/admin/` 独立管理入口 | 管整台服务器的网站备案资料；不等于某一家店主 |
| **买家** | 名录 / 店铺点菜页；野草生态登录 | 注册、点菜、下单、看自己的订单历史 |
| **店主（卖家）** | 野草生态登录 → 店铺管理；也可进本店工作台 | 管店、管员工账号、查操作留痕 |
| **服务员 / 后厨 / 骑手** | **只能**从本店工作台入口登录（扫码或链接） | 现场接单、备餐、配送；**进不了**店主后台 |

重要：

- **野草生态登录**（`/accounts/login/`）只给买家、店主。  
- **店铺工作台**（`/s/店铺码/work/`）给店主 + 本店员工，和生态登录是两套门，互不顶替。  
- **服务器拥有者管理入口**（`/admin/`）使用单独创建的超级管理员账号；即使还没有店铺，也可先在「网站合规信息」填写 ICP / 公安备案号。系统会自动使用官方查询链接并显示在公共页脚，普通店主不能修改。
- 公共电脑请勿勾选「记住密码」；关页后可能提示离开，并尽量退出登录。

## 3. 店主第一次怎么开起来（简要）

1. **开通店铺**（注册服务器）→ 系统店铺码。  
2. 用店主账号登录 → 进入**店铺管理**。  
3. 在「商品上架」加菜；在「堂食营业」设营业时段、桌台、桌码，需要时导出**桌贴 PDF** 打印贴桌。  
4. 在「员工工作台」创建服务员 / 后厨 / 骑手账号，打印或保存本店工作台二维码。  
5. 在「支付设置」按需要打开模拟支付、微信或现金（真微信开通见 `卖家微信支付开通指南.md`）。  
6. 员工用手机扫工作台二维码登录干活。

---

## 4. 买家怎么点

1. 打开店铺点菜页（或扫桌码进入堂食）。  
2. 选外卖 / 堂食 / 打包；把菜加入购物车后下单。  
3. 按页面提示付款；之后在「订单历史」查看进度。  

**会员价 / 特价**：一般需登录买家账号才可买；游客通常只能普通价（以店内实际展示为准）。

## 5. 堂食：桌码主单与拼桌（必读重点）

这两条是硬规则，店主后台「堂食营业」页也会醒目提示：

### 桌码主单（常用）

- **一桌一单**：客人扫桌上的码，开台后同一桌可继续加点，合并在同一主单。  
- 适合正常堂食。

### 拼桌单（默认关闭，慎开）

- 编号类似「桌号-临时号」；**全程靠服务员**；**不可加点合并**。  
- 开启前须阅读并确认拼桌硬规则；与桌码主单不是同一套玩法。

不清楚时：优先用桌码主单；拼桌仅在店主明确需要时开启。

## 6. 店铺工作台怎么用

入口示例：`/s/你的店铺码/work/`

登录后有三个视角（Tab）：

| 视角 | 常见操作 |
|------|----------|
| **服务员** | 看新单、按份标记上桌/交付、协助收款、堂食完成等 |
| **后厨** | 新单提醒、待备列表、按份备好、开始备货 |
| **骑手** | 接配送任务、取餐、送达 |

员工可切换**上班 / 休息 / 下班**（第一版已有基础；更完整的考勤表上线后再完善）。  
员工可点「我的操作记录」查看本人留痕；店主在店铺管理「操作留痕」可看本店更多记录与技术日志摘要。

---

## 7. 店主后台常用分区

| 分区 | 用途 |
|------|------|
| 订单管理 | **查历史**（搜索、筛选）；现场操作请到工作台 |
| 商品上架 | 菜品、菜单清单切换、销量排行等 |
| 堂食营业 | 营业/打烊/暂停、桌台桌码、拼桌、桌贴导出 |
| 员工工作台 | 二维码、三岗账号、外卖交接方式等 |
| 配送费规则 | 外卖配送费与减免 |
| 支付设置 | 模拟 / 微信 / 现金等 |
| 操作留痕 | 谁做了什么、技术日志摘要（排错与内控） |

切页或返回前若有未保存修改，系统会提示保存或放弃。

## 8. 登录与安全（用户须知）

- 约 **5 分钟**页面不报平安，或约 **15 分钟**无操作，可能自动退出。  
- 关掉标签页时，浏览器可能弹出离开提示；系统会尽量退出，**不保证每次都成功**。  
- 「记住用户名和密码」默认不勾；**无法 100% 禁止浏览器自己记密码**。  
- 离职员工请店主及时**停用**子账号。

更细的安全与部署原则见《野草系统-数据安全与隐私说明》。

## 9. 常见问题

**问：员工从名录页的「登录」进不去？**  
答：员工请用本店工作台链接或二维码，不要用野草生态登录。

**问：订单管理里不能接单？**  
答：订单管理只查历史；接单、备货、配送在**店铺工作台**。

**问：模拟支付和真微信有什么区别？**  
答：模拟是测试用「假装付过」；正式营业须用真微信并关闭模拟（上线前清查项）。

**问：退款怎么做？**  
答：系统内微信退款能力尚在上线前最后一批完成；当前以店内约定与支付渠道处理为准。

**问：名录页以后会变吗？**  
答：已变。打开网站根地址进入的是**服务器拥有者的展示主页**（积木拼成）。「名录」「野草介绍」都是可选块，由拥有者在后台决定开不开；旧地址 `/directory/` 会跳到入口主页。
